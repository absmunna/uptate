# PAIKARMART MASTER GLOBAL CONSTITUTION v1.0

## PROJECT IDENTITY

PaikarMart is not a single-purpose e-commerce platform.

PaikarMart is a long-term multi-tenant, multi-role, permission-based Super Ecosystem Platform designed for Bangladesh first and global expansion later.

The ecosystem may include:

* Commerce
* Social Commerce
* B2B Trade
* B2C Marketplace
* Wholesale
* Manufacturing
* Services
* Logistics
* Warehousing
* Food Delivery
* Travel
* Export
* Import
* Finance
* AI Automation

Every decision must support this vision.

---

# CORE PRINCIPLES

## Rule 1

Never design for one industry.

Always support:

* Products
* Services
* Logistics
* Food
* Travel
* Manufacturing
* Import/Export
* Future modules

---

## Rule 2

Never hardcode roles.

Always use:

```text
Account
→ Roles
→ Permissions
→ Business Profiles
```

Never:

```text
Customer Table
Seller Table
Factory Table
```

---

## Rule 3

One Account = Multiple Roles

Example:

```text
User
→ Customer
→ Seller
→ Factory Owner
→ Exporter
→ Service Provider
```

---

## Rule 4

Permission-Based Architecture

All access must be permission-driven.

Example:

```text
CAN_BUY
CAN_SELL
CAN_EXPORT
CAN_IMPORT
CAN_DELIVER
CAN_CREATE_SERVICE
CAN_CREATE_BUSINESS
CAN_MANAGE_STAFF
```

---

## Rule 5

Mobile First

All UI must support:

* Mobile
* Tablet
* Desktop
